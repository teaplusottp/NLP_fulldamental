## 1. Word Analogy là gì?

**Word Analogy** = bài toán kiểm tra **mối quan hệ giữa từ** bằng embeddings.  

- Ví dụ nổi tiếng:  

king - man + woman ≈ queen


- Công thức:

vec("king") - vec("man") + vec("woman") ≈ vec("queen")


- Mục tiêu: embeddings capture **semantic relationships**.

---

## 2. Các loại Semantic Relationships

1. **Gender relation**:  
   - king → queen  
   - man → woman  

2. **Country – Capital**:  
   - France → Paris  
   - Japan → Tokyo  

3. **Verb tense**:  
   - walk → walked  
   - go → went  

4. **Synonyms / Antonyms** (dễ hơn dùng similarity, nhưng vẫn áp dụng analogy)

---

## 3. Code Python – Word2Vec Analogy (gensim)

```python
from gensim.models import Word2Vec

sentences = [
    ["king", "queen", "man", "woman", "prince", "princess"],
    ["France", "Paris", "Japan", "Tokyo"]
]

model = Word2Vec(sentences, vector_size=50, window=2, min_count=1, sg=1)

# king - man + woman
result = model.wv.most_similar(positive=['king','woman'], negative=['man'], topn=1)
print(result)

5. Cosine Similarity vs Analogy
Cosine similarity: đo độ gần giữa 2 từ
Analogy: đo mối quan hệ vector → capture patterns giữa nhiều từ
6. Mini bài tập
Train Word2Vec → kiểm tra analogy: “tôi – học + đọc ≈ ?”
Dùng pretrained GloVe → test analogy: king – man + woman ≈ ?
Tìm 3 từ gần nghĩa nhất với “AI” bằng similarity

7. Lưu ý

Contextual embeddings (BERT, PhoBERT) → khó trực tiếp làm analogy, cần pooling vectors
