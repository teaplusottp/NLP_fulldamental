# 📘 Chương 1 - Bài 16: N-gram & Feature Engineering trong NLP

---

## 1. N-gram là gì?

**N-gram** = chuỗi gồm **N token liên tiếp** trong văn bản.

- 1-gram → unigram → từng từ đơn
- 2-gram → bigram → cặp từ
- 3-gram → trigram → bộ 3 từ

---

## 2. Tại sao dùng N-gram?

- Bắt được **ngữ cảnh / phrase**
- Giúp model hiểu **từ liền kề**
- Tăng khả năng **dự đoán & similarity**

---

## 3. Ví dụ trực quan

Text:

"Tôi thích học AI"


### 1-gram:

["Tôi", "thích", "học", "AI"]


### 2-gram:

["Tôi thích", "thích học", "học AI"]


### 3-gram:

["Tôi thích học", "thích học AI"]


---

## 4. Code Python – tạo N-gram

```python
from sklearn.feature_extraction.text import CountVectorizer

text = ["Tôi thích học AI"]

vectorizer = CountVectorizer(ngram_range=(1,2))
X = vectorizer.fit_transform(text)

print(vectorizer.get_feature_names_out())
```
Output:
['ai', 'học', 'thích', 'tôi', 'thích học', 'học ai', 'tôi thích']

5. Feature Engineering với N-gram
N-gram → feature trong:
Bag of Words
TF-IDF
Giúp:
Cải thiện accuracy
Bắt phrase quan trọng
6. Lưu ý khi dùng N-gram
N quá lớn → feature explosion → tốn bộ nhớ
N quá nhỏ → mất ngữ cảnh → underfitting
Thường dùng:
n=1,2 (unigram + bigram)
hiếm khi dùng n>3
7. Kết hợp với Stopwords & Vocabulary
Loại stopwords → giảm noise
Tạo vocabulary
Sinh N-gram → feature vector
8. Ứng dụng thực tế
Spam detection → “free money” → bigram
Sentiment analysis → “không tốt” → bigram
Search engine → phrase matching → bigram/trigram
Autocomplete → n-gram language model
