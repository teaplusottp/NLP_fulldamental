# 📘 Chương 1 - Bài 17: POS Tagging (Part-of-Speech Tagging) trong NLP

---

## 1. POS Tagging là gì?

**POS (Part-of-Speech) Tagging** = gán nhãn từ loại cho từng token trong câu:

- Danh từ (NOUN)  
- Động từ (VERB)  
- Tính từ (ADJ)  
- Trạng từ (ADV)  
- Đại từ, giới từ, liên từ,…

---

## 2. Ví dụ

Text:

"Tôi thích học AI"


POS Tagging:

| Token | POS |
|-------|-----|
| Tôi   | PRON |
| thích | VERB |
| học   | VERB |
| AI    | NOUN |

---

## 3. Tại sao POS quan trọng?

- Hiểu **ngữ pháp câu**
- Hỗ trợ:
  - Dependency parsing
  - Chunking
  - NER (Named Entity Recognition)
  - Information extraction

---

## 4. Phương pháp POS Tagging

### 🔹 4.1 Rule-based
- Dùng từ điển + pattern
- Ưu: nhanh, đơn giản
- Nhược: khó mở rộng, không chính xác với text tự do

### 🔹 4.2 ML-based
- Học từ corpus gán nhãn
- Ví dụ: HMM, CRF

### 🔹 4.3 Deep Learning
- BiLSTM + CRF
- Transformer (BERT-based) → state-of-the-art

---

## 5. Code Python – SpaCy (Nhanh & chuẩn)

```python
import spacy

nlp = spacy.load("en_core_web_sm")  # hoặc 'vi_core_news_sm' cho tiếng Việt

text = "Tôi thích học AI"
doc = nlp(text)

for token in doc:
    print(token.text, token.pos_)
Output:
Tôi PRON
thích VERB
học VERB
AI NOUN
6. POS Tagging trong tiếng Việt

7. Ứng dụng POS Tagging
Chunking / Phrase extraction: “Danh từ + Động từ”
Dependency Parsing: hiểu mối quan hệ từ → từ
NER cải thiện: POS giúp xác định entity chính xác
Information extraction: trích xuất facts từ text
8. Lưu ý khi dùng POS Tagging
Tokenization chuẩn → ảnh hưởng accuracy
Task-specific:
Search → POS ít quan trọng
NER / IE → POS quan trọng
Không phải từ nào cũng được gán đúng → cần domain-specific model
