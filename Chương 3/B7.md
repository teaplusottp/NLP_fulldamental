## 1. FastText là gì?

**FastText** (Facebook AI, Bojanowski et al., 2017) - word embedding model sử dụng **subword information** (character n-grams).

**Key innovation:** Xử lý được OOV words và rare words bằng cách decompose từng từ thành subwords.

---

## 2. Ý tưởng cốt lõi: Subwords

```python
word = "where"
# FastText chia thành character n-grams (n=3-6):
n_min, n_max = 3, 6
subwords = []
word_padded = f"<{word}>"  # thêm < > để mark boundaries

for n in range(n_min, n_max + 1):
    for i in range(len(word_padded) - n + 1):
        subwords.append(word_padded[i:i+n])

subwords.append(word_padded)  # thêm cả từ đầy đủ

print(f"FastText subwords for '{word}':")
for sw in subwords:
    print(f"  '{sw}'")
```

---

## 3. Tại sao Subwords quan trọng?

```
"running" → "<run", "run", "unn", "nni", "nin", "ing", "ng>", ...
"runs"    → "<run", "run", "uns", "ns>", ...

Subword "<run" và "run" shared! → "running" và "runs" có vectors gần nhau
```

**Xử lý OOV:**
```
"unbelievable" (OOV) → "un", "bel", "eli", "lie", "iev", "eva", ...
→ Tính từ average subword vectors → reasonable embedding!
```

## 4. Code với Gensim FastText

```python
from gensim.models import FastText
import numpy as np

sentences = [
    "I love running in the morning",
    "She runs every day for exercise",
    "The runner won the marathon race",
    "Running and jogging improve fitness",
    "I enjoy eating healthy food",
    "Healthy eating habits are important",
    "He eats vegetables every meal",
    "I study machine learning techniques",
    "Learning new programming skills is fun",
    "Deep learning models learn from data"
]

sentences_tokenized = [s.lower().split() for s in sentences]

# Train FastText
ft_model = FastText(
    sentences=sentences_tokenized,
    vector_size=100,
    window=5,
    min_count=1,
    sg=1,              # Skip-gram based
    min_n=3,           # min n-gram size
    max_n=6,           # max n-gram size
    workers=2,
    epochs=200,
    seed=42
)

print(f"Vocabulary size: {len(ft_model.wv)}")
print(f"Vector dimension: {ft_model.wv.vector_size}")
```

## 5. OOV Handling

```python
# OOV words có trong Word2Vec/GloVe → KeyError
# FastText: reconstruct từ subwords

print("\n--- OOV Handling ---")

# Known words
known_words = ["running", "learning", "eating"]
for word in known_words:
    vec = ft_model.wv[word]
    print(f"Known: '{word}' → vector norm = {np.linalg.norm(vec):.4f}")

# OOV words (không trong training vocab)
oov_words = ["runner", "runners", "ran", "unbelievable"]
for word in oov_words:
    try:
        vec = ft_model.wv[word]
        print(f"OOV: '{word}' → vector norm = {np.linalg.norm(vec):.4f} ✅")
    except KeyError:
        print(f"OOV: '{word}' → Not found ❌")

# Lưu ý: Gensim FastText có thể handle OOV qua subword
```

## 6. Morphological Similarity

```python
# FastText nắm tốt morphological relationships
print("\n--- Morphological Relationships ---")

# Các dạng của cùng một root
word_groups = [
    ["run", "running", "runs", "runner"],
    ["learn", "learning", "learned", "learner"],
    ["eat", "eating", "eats", "eaten"]
]

from sklearn.metrics.pairwise import cosine_similarity

for group in word_groups:
    print(f"\nGroup: {group}")
    for i, w1 in enumerate(group):
        for j, w2 in enumerate(group):
            if i < j:
                try:
                    v1 = ft_model.wv[w1].reshape(1,-1)
                    v2 = ft_model.wv[w2].reshape(1,-1)
                    sim = cosine_similarity(v1, v2)[0][0]
                    print(f"  {w1} vs {w2}: {sim:.4f}")
                except:
                    pass
```

## 8. FastText vs Word2Vec vs GloVe

| | Word2Vec | GloVe | FastText |
|-|----------|-------|---------|
| Unit | Word | Word | Character n-gram |
| OOV | ❌ Không | ❌ Không | ✅ Có |
| Rare words | Trung bình | Tốt | **Tốt nhất** |
| Morphology | ❌ | ❌ | ✅ |
| Speed | Nhanh | Trung bình | Chậm hơn |
| Tiếng Việt | Okay | Okay | **Tốt nhất** (nhiều dấu) |

---

## 9. Use Cases cho FastText

1. **Ngôn ngữ có morphology phức tạp**: tiếng Czech, Phần Lan, Việt...
2. **Tiếng lóng, typo**: "offiice" → gần "office"
3. **Medical/Legal**: rare technical terms
4. **Code**: programming keywords, camelCase

---

## 10. Tóm tắt

- FastText = Word2Vec + **subword information**
- Xây dựng embedding từ character n-grams
- **Giải quyết OOV** hoàn toàn
- Đặc biệt tốt với **morphologically rich languages**
- Với tiếng Việt: FastText > Word2Vec vì xử lý dấu/diacritics tốt hơn
