## 1. Feature Engineering là gì?

**Feature Engineering** là quá trình **biến dữ liệu thô thành các đặc trưng (features)** mà model có thể học được.

👉 Với NLP: đầu vào là text → phải convert thành số để ML hiểu được.

---

## 2. Tại sao cần Feature Engineering?

ML models chỉ làm việc với **vector số**, không hiểu text.

```
"Tôi yêu NLP" → [0.3, 0.1, 0.8, ...] (vector số)
```

Chất lượng features → ảnh hưởng trực tiếp đến độ chính xác của model.

---

## 3. Các loại features trong NLP

### 🔹 Lexical Features (từ vựng)
- Bag of Words
- TF-IDF
- N-gram
- Word count, char count

### 🔹 Syntactic Features (cú pháp)
- POS tags
- Dependency relations
- Parse tree features

### 🔹 Semantic Features (ngữ nghĩa)
- Word embeddings (Word2Vec, GloVe)
- Sentence embeddings

### 🔹 Statistical Features
- Sentence length
- Avg word length
- Number of stopwords
- Punctuation count

---

## 4. Pipeline Feature Engineering cho NLP

```
Raw Text
→ Cleaning (lowercase, remove noise)
→ Tokenization
→ Feature Extraction (TF-IDF, embeddings...)
→ Feature Selection (chọn lọc feature quan trọng)
→ Model Input
```

## 5. Ví dụ thực tế

### Bài toán: Phân loại spam

| Feature | Giá trị |
|---------|---------|
| word_count | 15 |
| has_url | 1 |
| exclamation_count | 3 |
| tfidf_free | 0.45 |
| tfidf_win | 0.38 |

→ Model dùng các features này để phán đoán spam/not spam.

---

## 6. Code ví dụ

```python
import pandas as pd
import re

texts = [
    "Bạn đã trúng thưởng 100 triệu! Click ngay!!!",
    "Họp nhóm lúc 3 giờ chiều nha",
    "FREE OFFER!!! Win $1000 now!!!",
    "Bài tập về nhà nộp sáng mai"
]

def extract_features(text):
    return {
        "word_count": len(text.split()),
        "char_count": len(text),
        "exclamation_count": text.count("!"),
        "uppercase_ratio": sum(1 for c in text if c.isupper()) / len(text),
        "has_number": int(bool(re.search(r'\d', text))),
        "has_url": int("http" in text or "www" in text or "click" in text.lower())
    }

df = pd.DataFrame([extract_features(t) for t in texts])
print(df)
```

## 7. Good vs Bad Features

| Feature tốt | Feature xấu |
|-------------|-------------|
| Có tương quan với label | Noise, không liên quan |
| Không bị leakage | Chứa thông tin từ tương lai |
| Scale hợp lý | Giá trị cực đoan |

---

## 8. Tóm tắt

- Feature engineering = biến text → số
- Chất lượng feature quyết định chất lượng model
- Deep Learning giảm bớt nhu cầu feature engineering thủ công
- Nhưng với ML cổ điển, feature engineering **cực kỳ quan trọng**
