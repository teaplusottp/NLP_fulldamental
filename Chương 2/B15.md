## 1. Imbalanced Data là gì?

Khi một class **áp đảo** các class khác:

```
Spam dataset:
  Not Spam: 950 samples  (95%)
  Spam:      50 samples  (5%)
→ Model ngốc predict "Not Spam" luôn: Accuracy = 95% 😱
```

Đây là vấn đề **cực kỳ phổ biến** trong NLP:
- Fraud detection: 99% normal, 1% fraud
- Medical NLP: 95% normal, 5% disease
- Hate speech: 90% clean, 10% toxic

---

## 2. Phát hiện Imbalanced Data

```python
from collections import Counter
import pandas as pd

labels = [0]*950 + [1]*50  # 950 normal, 50 spam

counts = Counter(labels)
print("Class distribution:")
for cls, count in counts.items():
    print(f"  Class {cls}: {count} ({count/len(labels)*100:.1f}%)")

# Imbalance ratio
imbalance_ratio = max(counts.values()) / min(counts.values())
print(f"Imbalance Ratio: {imbalance_ratio:.1f}:1")
```

## 3. Giải pháp 1: Class Weights

Cho model biết class minority quan trọng hơn:

```python
from sklearn.linear_model import LogisticRegression
from sklearn.svm import LinearSVC

# class_weight='balanced': tự tính weight dựa trên class frequency
lr = LogisticRegression(class_weight='balanced', max_iter=1000)

# Hoặc set thủ công
lr_custom = LogisticRegression(
    class_weight={0: 1, 1: 19},  # weight minority class 19x nặng hơn
    max_iter=1000
)

svm = LinearSVC(class_weight='balanced')
```

## 4. Giải pháp 2: Oversampling (SMOTE)

Tạo thêm samples cho class minority:

```python
# pip install imbalanced-learn
from imblearn.over_sampling import SMOTE, RandomOverSampler
from sklearn.feature_extraction.text import TfidfVectorizer

texts = ["spam sample " + str(i) for i in range(50)] + \
        ["normal text " + str(i) for i in range(950)]
labels = [1]*50 + [0]*950

vec = TfidfVectorizer()
X = vec.fit_transform(texts)

# Random Oversampling: duplicate minority samples
ros = RandomOverSampler(random_state=42)
X_resampled, y_resampled = ros.fit_resample(X, labels)
print(f"After RandomOverSampling: {Counter(y_resampled)}")

# SMOTE: tạo synthetic samples
smote = SMOTE(random_state=42, k_neighbors=3)
X_smote, y_smote = smote.fit_resample(X, labels)
print(f"After SMOTE: {Counter(y_smote)}")
```

---

## 5. Giải pháp 3: Undersampling

Giảm samples của class majority:

```python
from imblearn.under_sampling import RandomUnderSampler, TomekLinks

# Random Undersampling: xóa random samples majority
rus = RandomUnderSampler(random_state=42)
X_under, y_under = rus.fit_resample(X, labels)
print(f"After Undersampling: {Counter(y_under)}")
```

## 6. Giải pháp 4: Điều chỉnh Threshold

```python
from sklearn.metrics import precision_recall_curve
import numpy as np

# Train model với imbalanced data
lr = LogisticRegression(class_weight='balanced', max_iter=1000)
lr.fit(X, labels)

# Lấy probability
probs = lr.predict_proba(X)[:, 1]

# Tìm threshold tốt nhất dựa trên F1
precision, recall, thresholds = precision_recall_curve(labels, probs)
f1_scores = 2 * precision * recall / (precision + recall + 1e-10)
best_threshold = thresholds[np.argmax(f1_scores[:-1])]
print(f"Best threshold: {best_threshold:.3f}")

# Predict với custom threshold
y_pred_custom = (probs >= best_threshold).astype(int)
```

## 7. Metrics cho Imbalanced Data

**Đừng dùng Accuracy!** Dùng:

```python
from sklearn.metrics import (
    f1_score, roc_auc_score,
    average_precision_score, matthews_corrcoef
)

# F1 (weighted hoặc macro)
f1 = f1_score(labels, y_pred_custom, average='weighted')

# AUC-ROC
auc = roc_auc_score(labels, probs)

# Matthews Correlation Coefficient (MCC): tốt nhất cho imbalanced
mcc = matthews_corrcoef(labels, y_pred_custom)

# Average Precision (PR-AUC): tốt hơn ROC khi rất imbalanced
ap = average_precision_score(labels, probs)

print(f"F1: {f1:.4f}")
print(f"AUC-ROC: {auc:.4f}")
print(f"MCC: {mcc:.4f}")
print(f"AP (PR-AUC): {ap:.4f}")
```

## 8. So sánh Chiến lược

| Phương pháp | Ưu điểm | Nhược điểm |
|-------------|---------|------------|
| Class weights | Đơn giản, không thay đổi data | Ít điều chỉnh |
| Oversampling | Giữ toàn bộ data | Duplicate samples |
| SMOTE | Synthetic samples mới | Có thể overfit; cần dense space |
| Undersampling | Nhanh | Mất data valuable |
| Threshold tuning | Linh hoạt | Cần calibrated probabilities |

## 10. Tóm tắt

- Imbalanced = class phổ biến áp đảo → accuracy misleading
- **Luôn** dùng `class_weight='balanced'` làm baseline
- SMOTE hoạt động với dense features tốt hơn sparse TF-IDF
- Metric: dùng F1 (weighted), AUC-ROC, MCC
- Kiểm tra distribution data **trước** khi train
