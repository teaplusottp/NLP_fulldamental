## 1. Overfitting là gì?

**Overfitting** = model học quá tốt training data, **không tổng quát** được với data mới.

```
Train Accuracy:  98%
Test Accuracy:   62%
→ Overfit nặng!
```

---

## 2. Nguyên nhân Overfitting trong NLP

- **Quá ít dữ liệu** so với số features (TF-IDF có hàng nghìn dimensions)
- **Model quá phức tạp** (decision tree không giới hạn)
- **Training quá lâu** (nhiều epochs)
- **Features noise** (từ vựng không liên quan)

---

## 3. Phát hiện Overfitting

```python
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import accuracy_score
import numpy as np

# Generate synthetic data
np.random.seed(42)
n = 100
texts_train = [f"word{np.random.randint(0,20)} word{np.random.randint(0,20)}" for _ in range(n)]
labels_train = np.random.randint(0, 2, n).tolist()

texts_test = [f"word{np.random.randint(0,20)} word{np.random.randint(0,20)}" for _ in range(30)]
labels_test = np.random.randint(0, 2, 30).tolist()

vec = TfidfVectorizer()
X_train = vec.fit_transform(texts_train)
X_test = vec.transform(texts_test)

# Model không giới hạn depth → overfit
dt_overfit = DecisionTreeClassifier(max_depth=None, random_state=42)
dt_overfit.fit(X_train, labels_train)
print(f"No limit  - Train: {accuracy_score(labels_train, dt_overfit.predict(X_train)):.2f}, "
      f"Test: {accuracy_score(labels_test, dt_overfit.predict(X_test)):.2f}")

# Model giới hạn depth → ít overfit hơn
dt_limited = DecisionTreeClassifier(max_depth=5, random_state=42)
dt_limited.fit(X_train, labels_train)
print(f"max_depth=5 - Train: {accuracy_score(labels_train, dt_limited.predict(X_train)):.2f}, "
      f"Test: {accuracy_score(labels_test, dt_limited.predict(X_test)):.2f}")
```

## 4. Regularization L1 và L2

### L2 (Ridge / Weight Decay)

$$\mathcal{L} = \text{Loss} + \lambda \sum w_i^2$$

- Penalize large weights
- Weights nhỏ đều nhau
- sklearn: `penalty='l2'` (default)

### L1 (Lasso)

$$\mathcal{L} = \text{Loss} + \lambda \sum |w_i|$$

- Nhiều weights = 0 (sparse)
- Tự động feature selection
- sklearn: `penalty='l1'`

```python
from sklearn.linear_model import LogisticRegression

# L2 (default)
lr_l2 = LogisticRegression(penalty='l2', C=1.0, max_iter=1000)

# L1 (sparse weights - feature selection)
lr_l1 = LogisticRegression(penalty='l1', C=1.0, solver='saga', max_iter=1000)

# ElasticNet (kết hợp L1+L2)
lr_en = LogisticRegression(penalty='elasticnet', l1_ratio=0.5,
                            solver='saga', max_iter=1000)
```

## 5. Early Stopping

Dừng training khi validation score không tăng:

```python
from sklearn.linear_model import SGDClassifier
from sklearn.model_selection import train_test_split

X_tr, X_val, y_tr, y_val = train_test_split(X_train, labels_train,
                                               test_size=0.2, random_state=42)

best_score = 0
best_iter = 0

for i in range(1, 50):
    clf = SGDClassifier(loss='hinge', max_iter=i, warm_start=True,
                        random_state=42, tol=None)
    clf.fit(X_tr, y_tr)
    score = accuracy_score(y_val, clf.predict(X_val))
    if score > best_score:
        best_score = score
        best_iter = i

print(f"Best iteration: {best_iter}, Val Score: {best_score:.4f}")
```

## 6. Cross-Validation để detect Overfitting

```python
from sklearn.model_selection import cross_val_score
from sklearn.pipeline import Pipeline

pipeline = Pipeline([
    ('tfidf', TfidfVectorizer()),
    ('clf', DecisionTreeClassifier(max_depth=5, random_state=42))
])

cv_scores = cross_val_score(pipeline, texts_train, labels_train, cv=5)
print(f"CV Mean: {cv_scores.mean():.4f} ± {cv_scores.std():.4f}")
```

## 7. Các kỹ thuật chống Overfitting

| Kỹ thuật | Mô tả |
|----------|-------|
| Regularization | L1/L2 penalty |
| More data | Thu thập thêm samples |
| Data augmentation | Tạo data mới từ data gốc |
| Feature selection | Chỉ giữ features quan trọng |
| Ensemble | Random Forest, Boosting |
| Early stopping | Dừng trước khi overfit |
| Cross-validation | Detect sớm |

---

## 8. Tóm tắt

- Overfit: train tốt, test tệ
- Detect: so sánh train vs val/test score
- Fix: regularization, more data, simpler model
- Luôn evaluate trên **held-out test set** không dùng trong training
