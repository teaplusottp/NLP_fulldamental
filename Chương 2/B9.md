# 📗 Chương 2 - Bài 9: Bias vs Variance

---

## 1. Bài toán tổng quát trong ML

Khi train model, ta muốn model vừa **học tốt từ data** vừa **tổng quát hóa** tốt với data mới.

Có 2 nguồn lỗi chính: **Bias** và **Variance**.

---

## 2. Bias là gì?

**Bias** = lỗi do model **quá đơn giản**, không capture được pattern.

```
Training data:         ●  ●  ○  ●  ○  ○  ●

Model (high bias):     ─────────────────────  ← đường thẳng không fit
```

→ Model underfitting: lỗi cả trên train lẫn test

**Ví dụ NLP:**
- Dùng unigram BoW cho bài toán cần hiểu ngữ nghĩa
- Model quá đơn giản → miss các pattern phức tạp

---

## 3. Variance là gì?

**Variance** = lỗi do model **quá phức tạp**, học thuộc lòng training data.

```
Training data:         ●  ●  ○  ●  ○  ○  ●

Model (high variance): ≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋≋  ← zigzag theo data
```

→ Model overfitting: tốt trên train, tệ trên test

**Ví dụ NLP:**
- Decision Tree không giới hạn depth → học thuộc từng sentence
- Model ghi nhớ training data, không generalize

---

## 4. Bias-Variance Tradeoff

$$\text{Total Error} = \text{Bias}^2 + \text{Variance} + \text{Irreducible Noise}$$

```
            Low Variance   High Variance
Low Bias:   ✅ Ideal        ⚠️ Overfit
High Bias:  ⚠️ Underfit     ❌ Worst case
```

## 5. Nhận biết qua Learning Curves

```python
from sklearn.model_selection import learning_curve
import matplotlib.pyplot as plt
import numpy as np
from sklearn.svm import LinearSVC
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.pipeline import Pipeline

# Pipeline: TF-IDF → SVM
pipeline = Pipeline([
    ('tfidf', TfidfVectorizer()),
    ('clf', LinearSVC(C=1.0))
])

# Dữ liệu mẫu lớn hơn
import random
texts = ["positive review " + str(i) for i in range(50)] + \
        ["negative review " + str(i) for i in range(50)]
labels = [1]*50 + [0]*50

train_sizes, train_scores, val_scores = learning_curve(
    pipeline, texts, labels,
    train_sizes=np.linspace(0.1, 1.0, 10),
    cv=5, scoring='accuracy'
)

plt.figure(figsize=(10, 6))
plt.plot(train_sizes, train_scores.mean(axis=1), label='Train score')
plt.plot(train_sizes, val_scores.mean(axis=1), label='Val score')
plt.xlabel('Training size')
plt.ylabel('Accuracy')
plt.title('Learning Curves')
plt.legend()
plt.grid(True)
plt.show()
```

---

## 6. Đọc Learning Curves

### High Bias (Underfitting):
```
Train score:  0.75 ─────────
Val score:    0.73 ─────────
```
→ Cả hai đều thấp và gần nhau → cần model phức tạp hơn

### High Variance (Overfitting):
```
Train score:  0.99 ─────────
Val score:    0.72 ──────────  (nhiều dưới train score hơn)
```
→ Train cao, val thấp → cần regularization hoặc nhiều data hơn

---

## 7. Giải pháp

### Nếu High Bias:
- Tăng model complexity (nhiều features, deep model)
- Thêm features (ngram, embeddings)
- Giảm regularization

### Nếu High Variance:
- Thêm dữ liệu training
- Tăng regularization (L1/L2, dropout)
- Giảm model complexity
- Dùng ensemble methods

---

## 8. Trong NLP cụ thể

| Situation | Bias/Variance | Solution |
|-----------|---------------|----------|
| BoW không capture meaning | High Bias | Thêm n-gram, embeddings |
| DT overfit | High Variance | Prune tree, Random Forest |
| SVM C quá lớn | High Variance | Giảm C |
| LR underfitting | High Bias | Thêm features, tăng C |

---

## 9. Tóm tắt

- **Bias**: model quá đơn giản → underfitting
- **Variance**: model quá phức tạp → overfitting
- **Goal**: tìm điểm cân bằng tối ưu
- Learning curves là tool quan trọng để diagnose
- Hầu hết models NLP thực tế bị về bias (thiếu data/features)
