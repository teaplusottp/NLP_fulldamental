## 2. ML NLP Decision Flowchart

```
New NLP Classification Task
         ↓
[1] Explore & understand data
    - Class distribution
    - Text length distribution  
    - Sample inspection
         ↓
[2] Preprocess text
    - Lowercase, clean
    - Tokenize
    - Remove stopwords (optional)
         ↓
[3] Vectorize
    - Try TF-IDF bigram first
    - Check vocabulary size
         ↓
[4] Split data
    - train/val/test (60/20/20)
    - Stratified split
         ↓
[5] Baseline model
    - Logistic Regression / Naive Bayes
    - 5-fold StratifiedCV
    - Report F1-weighted
         ↓
[6] Compare models
    - LR, NB, SVM, RF
    - Same CV setup
         ↓
[7] Handle issues
    - Imbalanced? → class_weight='balanced'
    - Overfit? → regularization, less features
    - Underfit? → more features, complex model
         ↓
[8] Hyperparameter tuning
    - GridSearch / RandomSearch
    - Tune on val, NOT test
         ↓
[9] Final evaluation on TEST SET (once!)
    - Report all metrics
    - Confusion matrix
         ↓
[10] Error analysis
    - What kind of errors?
    - Any patterns?
         ↓
[11] Deploy & monitor
```

## 5. Common Mistakes và cách tránh

| Mistake | Hậu quả | Fix |
|---------|---------|-----|
| Fit vectorizer trên all data | Data leakage | Dùng Pipeline |
| Dùng Accuracy khi imbalanced | Misleading metrics | Dùng F1 weighted |
| Thay đổi nhiều thứ cùng lúc | Không biết cái nào quan trọng | Change one thing at a time |
| Tune trên test set | Overoptimistic results | Tune trên val, test chỉ 1 lần |
| Không làm error analysis | Miss important insights | Error analysis sau mỗi iteration |
| Skip cross-validation | Unstable estimates | Always use CV |

## 6. Kiến thức sang Chương 3

Chương 2 học ML classical với **sparse, manual features (TF-IDF)**.

Vấn đề còn lại:
- ❌ "good" và "great" hoàn toàn độc lập
- ❌ "bank" (ngân hàng) = "bank" (bờ sông) trong BoW
- ❌ Không capture semantic meaning
- ❌ OOV: từ mới không có trong vocab = không được xử lý

Chương 3 sẽ giải quyết với **Word Embeddings**:
- Word2Vec, GloVe, FastText
- Semantic vectors: "good" ≈ "great" trong embedding space
- Context-aware (Contextual embeddings)
- Xử lý OOV với subword models

## 7. Tóm tắt Final

```
Classical ML cho NLP = 
   TF-IDF (features) + 
   LR/SVM/NB (model) + 
   Pipeline (no leakage) + 
   CV (robust evaluation) + 
   GridSearch (tuning) + 
   Error Analysis (improvement)
```

**Khi nào Classical ML là đủ:**
- Dataset < 50k samples
- Thời gian train quan trọng
- Interpretability required
- Baseline cần nhanh

**Khi nào cần Deep Learning:**
- Large dataset
- Complex semantics
- SOTA performance needed
- Resources available
